Q

Git（master | REBASE 1/1）是什么意思？我如何摆脱它？

  • version-control
  • github
  • branch
  • git-branch
  • rebase
2013-08-17 3310 views 7 likes 
7

我是一个git初学者，我试图回滚到以前的提交。但我不小心只是回滚提交（我正在使用Windows图形用户界面）。无论如何，经过一些奇怪的推动，合并和其他令人困惑的东西，我不太明白，我最终以我想要的方式获得了我的文件。唯一奇怪的是在shell现在它说：Git（master | REBASE 1/1）是什么意思？我如何摆脱它？

（主| REBASE 1/1）

它曾经只是说master，所以发生了什么事？这是什么意思？我怎样才能回到它是如何？

来源

2013-08-17 Captain Stack

A

回答

9

你是停留在一个变基的中间，你已经合并了所有的路径，使用git rebase --continue完成该过程

来源

2013-08-17 19:41:33 dkrikun

2

如果git rebase --continue | --skip | --abort仍不起作用：

您可以尝试放弃你本地提交一个接一个，然后使用git status确保您的本地提交与远程分支保持同步。

git reset --hard HEAD~1

注：git reset --hard将放弃你的工作，只使用它，如果你知道你在做什么！
